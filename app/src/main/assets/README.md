# AutoGLM Android 助手使用文档

欢迎使用 **AutoGLM Android 助手**！这是一个基于多模态大模型的智能代理应用，可以帮助您自动化操作手机完成各种任务。

## 1. 快速开始

### 第一步：授予必要权限
首次打开应用时，系统会提示您授予以下两个关键权限，这是应用正常运行的基础：
*   **无障碍服务权限 (Accessibility Service)**：用于让 AI 模拟点击、滑动和读取屏幕内容。
*   **悬浮窗权限 (Overlay Permission)**：用于在其他应用上方显示 AI 的运行状态和控制面板。

请点击应用内的提示按钮（如“去开启”、“去授权”），跳转到系统设置中开启对应开关。

### 第二步：获取并配置 API Key
本应用基于智谱 AI 的大模型能力，需要配置 API Key 才能使用。

1.  **获取 API Key**：
    *   访问 [智谱AI开放平台 - API Key管理](https://bigmodel.cn/usercenter/proj-mgmt/apikeys)。
    *   登录您的账号（支持手机号或微信扫码）。
    *   在页面中点击“创建 API Key”或复制已有的 Key。

2.  **填入应用设置**：
    *   回到 AutoGLM App。
    *   点击右上角的 **设置 (Settings)** 图标。
    *   点击 **"填入 API Key"** 或 **"修改 API Key"**。
    *   将复制的 API Key 粘贴到输入框中并保存。

### 第三步：开始使用
1.  在首页的对话框中输入您的指令，例如：“帮我下单一份肯德基的翅桶套餐”、“打开抖音，搜索喜德胡AI编程，并给他的视频点赞”。
2.  点击 **发送** 按钮。
3.  应用会自动跳转到桌面或相应应用，并通过**悬浮窗**展示当前的执行状态。

---

## 2. 悬浮窗状态说明

在任务执行过程中，屏幕右下角会出现一个悬浮窗，用于展示 AI 的当前状态。

### 状态详解

**1. AutoGLM 就绪 (绿色)**
*   **含义**：表示当前空闲，等待指令。
*   **操作**：点击 **"返回应用"** 按钮可回到主界面输入新指令。

**2. AutoGLM 运行中 (灰色)**
*   **含义**：表示 AI 正在思考或执行动作（如“思考中...”、“点击屏幕”）。
*   **操作**：**请勿手动操作屏幕**，以免干扰 AI。如需强行中断，可点击悬浮窗上的 **"停止"** 按钮。

**3. AutoGLM 出错 (红色)**
*   **含义**：表示任务执行遇到问题（如超时、无法识别元素）。
*   **操作**：您可以点击 **"停止"** 或 **"返回应用"** 查看具体报错信息。

### 常见状态文本
*   **思考中...**：模型正在规划下一步操作。
*   **点击屏幕 / 滑动屏幕**：正在执行具体的触控操作。
*   **等待中**：正在等待页面加载。
*   **已停止**：任务被用户手动取消或自然结束。

## 3. 常见问题
*   **为什么不执行动作？** 请检查无障碍服务是否被系统关闭（部分手机清理后台时会误关）。
*   **API Key 无效？** 请确认 Key 是否有多余空格，或账户余额是否充足。

祝您使用愉快！

## 5. 开发者配置 (For Developers)

如果您是开发者并希望在本地配置默认 API Key，请在项目根目录的 `local.properties` 文件中添加以下内容：

```properties
ZHIPU_API_KEY=your_actual_api_key_here
```

这样做的好处是 `local.properties` 默认被 git 忽略，不会将您的 Key 提交到代码仓库中。应用启动时会优先读取本地存储的 Key，如果没有则读取此配置作为默认值。
